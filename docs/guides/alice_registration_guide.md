# Alice新用户注册OraSRS网络完整流程

## 概述
本文档详细描述了新用户Alice如何下载OraSRS客户端、获取Gas补贴、领取启动资金并完成节点注册的完整流程。

## 步骤1：下载和安装客户端

Alice访问OraSRS官方网站：
```
https://api.orasrs.net/download
```

根据她的操作系统（Windows/Mac/Linux）下载相应的客户端安装包。安装完成后，客户端会自动配置连接参数以连接到OraSRS协议链。

## 步骤2：客户端初始化

客户端启动时：
1. 自动生成一个新的钱包地址
2. 钱包初始状态：没有原生代币（ETH/BNB等）也没有ORA代币
3. 客户端自动配置连接参数：
   - API端点: https://api.orasrs.net
   - Chain ID: 8888

## 步骤3：自动获取Gas补贴

### 问题识别
客户端检测到钱包余额为0，无法支付交易费用。

### 请求Gas补贴
客户端自动调用API端点：
```
POST /api/gas-subsidy/request
```

请求参数：
```json
{
  "userAddress": "0x...",
  "captchaToken": "...",
  "ip": "..."
}
```

### 后端验证和处理
1. 服务器验证Alice的身份（通过验证码、IP限制等）
2. 验证通过后，治理服务器调用GasSubsidy合约：
   ```
   gasSubsidy.connect(relayer).subsidize('0x...')
   ```
3. GasSubsidy合约地址：`0x5FC8d32690cc91D4c39d9d3abcBD16989F875707`
4. Alice钱包收到1个原生代币作为Gas补贴，用于支付后续交易费用

## 步骤4：领取启动资金

### 访问水龙头合约
Alice在客户端界面点击"获取启动资金"按钮，客户端调用FaucetUpgradeable合约：
```
faucetContract.withdrawTokens()
```

Faucet合约地址：`0x9A9f2CCfdE556A7E9Ff0848998Aa4a0CFD8863AE`

### 交易执行
1. 交易费用由刚刚获得的Gas补贴支付
2. Alice获得1000 ORA代币作为启动资金
3. 这些代币可用于节点质押或参与网络治理

## 步骤5：节点注册

### 配置节点参数
Alice在客户端界面配置节点参数：
- IP地址: 自动检测或手动输入（如 192.168.1.100）
- 端口: 8080（默认）
- 节点类型: 边缘层威胁检测节点
- 节点名称: Alice's Security Node

### 提交注册交易
点击"注册节点"按钮后，客户端生成并发送交易到NodeRegistry合约：
```
nodeRegistry.registerNode('192.168.1.100', 8080)
```

NodeRegistry合约地址：`0xc6e7DF5E7b4f2A278906862b61205850344D4e7d`

交易费用由Gas补贴支付，成功注册为OraSRS网络节点。

## 步骤6：节点激活和运行

注册成功后：
1. Alice的节点被添加到网络节点列表
2. 节点开始执行以下任务：
   - 监控网络流量，检测潜在威胁
   - 验证其他节点提交的威胁情报
   - 参与共识验证过程
   - 与其他节点同步威胁情报

## 收益机制

Alice开始获得代币奖励：
- 威胁检测奖励：每次成功检测和报告威胁
- 验证奖励：参与威胁情报验证
- 质押奖励：通过质押ORA代币获得
- 治理奖励：参与网络治理决策

## 网络贡献

Alice的节点成为OraSRS三层架构的一部分：
- 边缘层：负责实时威胁检测和本地响应
- 共识层：验证威胁情报并确保数据一致性
- 智能层：协调威胁情报并与其他安全系统集成

## 安全与合规

所有操作都符合当地法律法规：
- 威胁数据经过隐私保护处理
- 支持国密算法（SM2/SM3/SM4）和国际标准
